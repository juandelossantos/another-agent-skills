## When to Use What

| User Says | First Action |
|---|---|
| "Build X" | Load `spec-driven-development` → Discovery → SPEC.md |
| "Fix bug" | Load `debugging-and-error-recovery` → Repro test → Fix |
| "Review code" | Load `project-health-check` + `code-review-and-quality` |
| "Brainstorm" | Load `idea-refine` + `architecture-analysis` |
| "Make it faster" | Load `performance-optimization` → Profile → Fix |

---

## Quick Anti-Rationalization

**Wrong:** "I'll just quickly implement this." → **Right:** Check skills first.
**Wrong:** "Turbo means skip everything." → **Right:** Skip optional phases only.
**Wrong:** "User already said yes." → **Right:** Every commit is separate.
