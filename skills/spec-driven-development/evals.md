# Evals: spec-driven-development

Pass/fail checks to verify the skill's output meets quality standards.

## Spec Quality Checks

| # | Check | Pass Criteria |
|---|---|---|
| 1 | **SPEC.md exists** | File created with objective, scope, stack, acceptance criteria |
| 2 | **Objective clear** | One sentence describing what to build |
| 3 | **Scope defined** | In-scope and out-of-scope items listed |
| 4 | **Stack specified** | Technologies and versions listed |
| 5 | **Acceptance criteria** | At least 3 testable criteria |
| 6 | **Boundaries defined** | What NOT to build is explicit |
| 7 | **Edge cases** | At least 2 edge cases identified |
| 8 | **Dependencies listed** | External dependencies noted |
| 9 | **No ambiguity** | No "TBD", "maybe", or vague language |
| 10 | **User confirmed** | User explicitly approved the spec |

## Scoring

- 8-10 checks pass: Spec is complete and actionable
- 5-7 checks pass: Spec needs more detail
- <5 checks pass: Spec is insufficient
