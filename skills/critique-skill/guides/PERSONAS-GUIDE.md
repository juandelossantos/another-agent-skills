# Critique — Personas

## The Evaluator

> Comparing this to alternatives. Usually on a Tuesday evening.

| Attribute | Detail |
|---|---|
| **Mindset** | Comparative, skeptical, task-driven |
| **Device** | Desktop, multiple tabs open |
| **Time** | 5-10 minutes to evaluate |
| **Needs** | See value proposition immediately. Compare features. Trust signals. |
| **Friction** | Vague claims, no pricing, slow load, no social proof |
| **Score focus** | 1 (Visibility), 2 (Real world), 5 (Error prevention) |

Questions to ask through this lens:
- Can they understand what this does in 5 seconds?
- Can they compare with alternatives without leaving?
- Do they trust this enough to try it?

## The Returning User

> Knows the product. On mobile. In a hurry.

| Attribute | Detail |
|---|---|
| **Mindset** | Impatient, goal-oriented, familiar |
| **Device** | Mobile (375pt-428pt viewport) |
| **Time** | 30 seconds to complete task |
| **Needs** | Quick access to core action. Minimal navigation. Remembered state. |
| **Friction** | Re-authentication, reset context, slow load, changed layouts |
| **Score focus** | 3 (User control), 6 (Recognition), 7 (Efficiency) |

Questions to ask through this lens:
- Can they complete their most common task in 3 taps?
- Does the interface remember their preferences?
- Is the mobile experience as fast as the desktop?

## The Skeptic

> Has seen every SaaS landing page. Is bored.

| Attribute | Detail |
|---|---|
| **Mindset** | Cynical, pattern-aware, easily dismissed |
| **Device** | Desktop or tablet |
| **Time** | 15 seconds to decide if this is different |
| **Needs** | Surprise them. Break the pattern. Earn attention. |
| **Friction** | Same hero layout, same three cards, same testimonials carousel |
| **Score focus** | 8 (Aesthetic), 2 (Real world), 9 (Help errors) |

Questions to ask through this lens:
- Does this look like every other SaaS? (If yes, you lost them.)
- Is there a single moment that surprises or delights?
- Does the copy sound like a person wrote it?

## The Power User

> Needs speed. Knows what they want. Won't read instructions.

| Attribute | Detail |
|---|---|
| **Mindset** | Expert, efficiency-driven, low patience for UI |
| **Device** | Desktop with large monitor |
| **Time** | As fast as possible |
| **Needs** | Keyboard shortcuts, bulk actions, minimal clicks, visible data |
| **Friction** | Forced tutorials, slow animations, hidden data, paginated lists |
| **Score focus** | 7 (Flexibility), 5 (Error prevention), 10 (Help) |

Questions to ask through this lens:
- Can they complete a task without touching the mouse?
- Are bulk actions available for repetitive tasks?
- Is there an expert mode or settings to strip the UI?

## Usage

Each persona gets a score 1-4:

| Score | Meaning |
|---|---|
| 1 | This persona would **abandon** the interface |
| 2 | This persona would **struggle** but complete their task |
| 3 | This persona would be **satisfied** |
| 4 | This persona would be **delighted** |

Average all 4 persona scores. If any single score is 1, flag as P0.
