# Divergent and Convergent Techniques

> **Sources:** Design Council Double Diamond (designcouncil.org.uk) — diverging to explore, converging to focus. IDEO Design Kit (designkit.org) — human-centered ideation methods. De Bono's Six Thinking Hats — parallel thinking for evaluation.

## Divergent Techniques (Expand Options)

### SCAMPER

| Letter | Lens | Question |
|---|---|---|
| S | Substitute | What could we replace? |
| C | Combine | What could we merge? |
| A | Adapt | What else is like this? |
| M | Modify | What if we magnify/minimize? |
| P | Put to other uses | Who else could use this? |
| E | Eliminate | What if we removed this? |
| R | Reverse | What if we did the opposite? |

### Reverse Thinking

List every assumption. Flip each one. Ask: "What if the opposite were true?" Best for incremental ruts — forces genuinely different directions.

### First Principles

1. State the current solution or assumption.
2. Decompose to fundamental truths (what's physically/biz-logically true?).
3. Identify which parts are inherited conventions, not fundamental.
4. Rebuild from fundamentals.

### How Might We (HMW)

Bad: "How might we add a button for X?"
Good: "How might we help users accomplish Y without requiring Z?"

Structure: "How might we [desired outcome] for [user] without [constraint]?"

### Worst Possible Idea

Deliberately suggest terrible ideas. Lowers inhibition. Often the inversion of a terrible idea reveals a good one.

## Convergent Techniques (Select Path)

### Impact/Effort Matrix

Place each idea on a 2x2 grid:

| | High Impact | Low Impact |
|---|---|---|
| **Easy** | Do first | Do if trivial |
| **Hard** | Worth risk | Skip |

### NUF Test

Score each idea 1-10 on:
- **N**ovelty — How new/different is it?
- **U**sefulness — How valuable would it be?
- **F**easibility — How achievable is it?

Total score = priority ranking. Minimum threshold per dimension: 5.

### Assumption Audit

For the selected direction, list:
- **Must Be True** (dealbreakers if wrong)
- **Should Be True** (important but not fatal)
- **Might Be True** (nice to have)

Focus validation efforts on Must Be True items first.

### Pre-mortem

Imagine it's 12 months later and the project failed catastrophically. Write down every reason why. Work backwards: which of these risks can be mitigated now?

## Facilitation Script

```
1. OPEN: "Here's what I understand you want. Before we spec it, let's explore alternatives."
2. DIVERGE (10-15 min): Apply 2-3 divergent techniques. Generate 3+ distinct directions.
3. PAUSE: "Let's list what we have. Any obvious standouts? Any obvious dealbreakers?"
4. CONVERGE (10-15 min): Apply impact/effort + assumption audit.
5. SELECT: "Based on this, direction [X] scores highest. Here's why."
6. VERIFY: "Does this direction feel right? Any angle we missed?"
```

Never skip Step 5 rationale. "Felt right" is not a decision — score each option explicitly.
