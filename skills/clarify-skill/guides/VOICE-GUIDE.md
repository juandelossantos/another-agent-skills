# Clarify Skill — Voice Tuning

Adjust voice based on audience from `PRODUCT.md`. If no `PRODUCT.md` exists, detect audience from context.

## Audience Profiles

| Audience | Example | Register |
|---|---|---|
| Technical (developers) | "Invalid JSON at line 14. Check trailing comma." | Precise jargon, direct |
| Consumer | "This doesn't look like a valid email. Try name@company.com." | Plain language, helpful |
| Rushed (mobile, mid-task) | "Connection lost. Tapping retry." | Short, direct, minimal |
| Anxious (payment, delete) | "Your card won't be charged until you confirm." | Reassurance, explicit |
| Regulated (healthcare, finance) | "Session expired for security. Please re-authenticate." | Formal, complete |

## Principles by Audience

- **Technical:** Trust the user's knowledge. Be precise, not patronizing.
- **Consumer:** Assume no domain knowledge. Be warm, guide every step.
- **Rushed:** Strip every unnecessary word. One glance should tell the story.
- **Anxious:** Over-explain consequences. Give control back to the user.
- **Regulated:** Follow compliance requirements first. Clarity second.
