The 6-Phase Lifecycle

Every project follows the same disciplined flow. No shortcuts. No skipped steps.

Visual Flow

DEFINE → PLAN → BUILD → VERIFY → REVIEW → SHIP

Phase 1: DEFINE

Goal: Write the spec before any code.

Skills

  • spec-driven-development — Research-backed specs with critical thinking
  • architecture-analysis — 2-3 options evaluated with trade-offs
  • interview-me — Extract what the user actually wants
  • idea-refine — Refine vague ideas into sharp concepts

Output: SPEC.md, DESIGN.md, .gitignore

Gate: No code until contract exists (Rule 7).

Phase 2: PLAN

Goal: Break work into atomic tasks.

Skills

  • planning-and-task-breakdown — Decompose into verifiable chunks

Output: Task list with acceptance criteria

Phase 3: BUILD

Goal: Implement one slice at a time.

Skills

  • incremental-implementation — Thin vertical slices
  • test-driven-development — Failing test first, then make it pass
  • source-driven-development — Verify against official docs
  • doubt-driven-development — Adversarial review of non-trivial decisions

Output: Working code with tests

Gate: Each slice tested before expanding.

Phase 4: VERIFY

Goal: Run tests. Check builds. Verify behavior.

Skills

  • test-driven-development — Run test suite
  • debugging-and-error-recovery — Systematic root-cause debugging
  • browser-testing-with-devtools — Runtime verification

Output: Passing tests, clean build

TOOL_GAP: If tools can't reach the world, report "ship status unknown." Never fake a win.

Phase 5: REVIEW

Goal: Code review. Security. Performance. Quality.

Skills

  • code-review-and-quality — Five-axis review with severity labels
  • security-and-hardening — OWASP prevention
  • performance-optimization — Measure first, optimize only what matters

Output: Review report, quality gates passed

Design Review Pipeline (if UI work): critique → audit → clarify → hard → polish → typeset → adapt → optimize → delight

Phase 6: SHIP

Goal: Clean commits. CI/CD. Deploy.

Skills

  • git-workflow-and-versioning — Atomic commits, clean history
  • ci-cd-and-automation — Automated quality gates
  • shipping-and-launch — Pre-launch checklist, monitoring, rollback

Output: Deployed code, documentation updated

Purpose-Driven Execution

The lifecycle weights phases by session purpose:
