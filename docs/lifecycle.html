The 6-Phase Lifecycle

Every project follows the same disciplined flow. No shortcuts. No skipped steps.

Visual Flow

DEFINE → PLAN → BUILD → VERIFY → REVIEW → SHIP

DEFINE

Write the spec before any code.

Skills

  • spec-driven-development — Research-backed specs with critical thinking
  • architecture-analysis — 2-3 options evaluated with trade-offs

Output: SPEC.md, DESIGN.md, .gitignore

Gate: No code until contract exists (Rule 7).

PLAN

Break work into atomic tasks.

BUILD

Implement one slice at a time.

Gate: Each slice tested before expanding.

VERIFY

Run tests. Check builds. Verify behavior.

TOOL_GAP: If tools can't reach the world, report "ship status unknown." Never fake a win.

REVIEW

Code review. Security. Performance. Quality.

SHIP

Clean commits. CI/CD. Deploy.
