OpenCode (Recommended)

No extra setup. Skills load automatically via the skill tool.

bash install.sh          # Installs skills globally
cd your-project
init-agents              # Activates in this project

Claude Code

bash install.sh --agent claude

Creates .claude-plugin/agent-discipline/ with hooks. To also load SOUL.md and AGENTS.md rules:

# Copy rules into Claude's config
cp SOUL.md .claude/CLAUDE.md.append
cp AGENTS.md .claude/CLAUDE.md.append

Key principles to add to CLAUDE.md:

  • Rule 0h: TOOL_GAP
  • Rule 0i: Continuation Over Recap
  • Principle 8: Verification without evidence is inspection
  • Severity labels: blocking → important → nit → suggestion → learning → praise

Cursor

bash install.sh --agent cursor

Creates .cursor-plugin/agent-discipline/ with hooks. To load rules:

cat SOUL.md AGENTS.md >> .cursorrules

Devin Desktop

Formerly Windsurf. Acquired by Cognition Labs in 2025.

bash install.sh --agent kiro

Creates .kiro/hooks/agent-discipline.json with pre-flight, commit approval, and edit guard hooks.

Other Agents

GitHub Copilot, Gemini CLI, and any git-based agent: Copy the hooks to your project:

cp scripts/git-hooks/pre-commit .git/hooks/pre-commit
cp scripts/git-hooks/commit-msg .git/hooks/commit-msg
chmod +x .git/hooks/pre-commit .git/hooks/commit-msg
cp scripts/approve-commit.sh scripts/

Using Principles in Your Own System

The core principles are agent-agnostic. Adopt them anywhere:

PrincipleHow to Use
TOOL_GAPWhen tools can't verify, report "ship status unknown." Never fake success.
Severity LabelsClassify findings: blocking → important → nit → suggestion → learning → praise.
Error Path DesignEvery tool call, gate, and loop needs a failure path.
Continuation Over RecapAfter context loss, resume. Don't waste tokens re-explaining.
Drift DetectionCheck docs vs reality regularly.
Manifest GateRequire a written summary before commit approval.
