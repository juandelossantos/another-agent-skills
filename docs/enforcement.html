Enforcement

Mechanical enforcement, not suggestions.

Four Levels of Enforcement

Level 1: Process Rules

Agent remembers rules from AGENTS.md. Load-bearing for simple tasks, but fails when attention drifts.

Level 2: Visible Manifest (Guardian Pattern)

Before any mutation (commit, push, merge, rebase), the agent must present a DECISION POINT block, explain rationale, and wait for explicit approval. Invalid responses like "ok" and "continue" are rejected.

Level 3: SHA256 Hash-Bound Token

The commit-msg hook verifies that a SHA256 token matches the commit message. The agent cannot bypass what it cannot forge. Token is deleted after each commit.

Level 4: Manifest Gate

Before a token can be generated, a manifest file must exist at .git/COMMIT_MANIFEST. The agent MUST write this file before running approve-commit.sh. This prevents generating tokens without presenting changes.

The flow:

  1. Agent writes manifest (files, changes, impact, risk)
  2. Agent presents manifest to user
  3. User approves
  4. Agent generates token
  5. Git commit validates token, deletes it

Pre-Commit Gates (v7)

The pre-commit hook runs 8 checks before every commit:

GateWhat it checks
Branch checkNot on detached HEAD
Staged changesSomething is staged
Remote syncNo unpulled remote changes
HTML integrityStaged HTML files have required structural markers
Skill gateSkills were consulted before implementation
Build verificationShell scripts have valid syntax
Anti-slopNo generic AI-generated patterns detected
Debug trackingFix/debug commits tracked, escalates at 3

Incident History

Three Rule 12 violations in 48 hours produced three levels of enforcement:

  • INCIDENT_001: Agent committed without approval → Added visible manifest requirement
  • INCIDENT_002: Agent bypassed manifest → Added SHA256 hash-bound tokens
  • INCIDENT_003: Agent generated tokens without real approval → Added mandatory manifest gate

Honest Limitations

Warning

These gates create friction, not guarantees. The hooks enforce git operations, but cannot prevent the agent from generating tokens without presenting the manifest, interpreting ambiguous responses as approval, or pushing without asking. The human must stay alert.
