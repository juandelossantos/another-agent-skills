Design Review Pipeline

The pipeline turns subjective design feedback into a deterministic, measurable process. Each skill handles one dimension.

The Pipeline

critique → audit → clarify → hard → polish → typeset → adapt → optimize → delight

9 Skills

1. critique-skill — Design Quality

Heuristic review using Nielsen's 10 usability heuristics and 4 persona tests. Detects AI slop patterns. Produces a score per heuristic.

Output: Score 0-4 per heuristic, list of findings

2. audit-skill — Technical Quality

5-dimension scan: accessibility, performance, theming, responsive, anti-patterns. P0-P3 severity scoring.

Output: Score 0-4 per dimension, severity-classified findings

3. clarify-skill — UX Copy

Rewrites labels, errors, buttons, empty states, confirmations. Voice-tuned per audience.

Output: Rewritten copy with 8 QA gates

4. hard-skill — Accessibility & Robustness

Mechanical P0/P1 fixes: ARIA attributes, keyboard navigation, form validation, error/empty/loading states.

Output: Fixed code, traces to audit findings

5. polish-skill — Design Details

Fixes spacing, alignment, token drift, border radius, shadows. No design decisions — only compliance.

Output: Token-compliant styles

6. typeset-skill — Typography

Applies type ramp, fixes line-height, letter-spacing, paragraph rhythm.

Output: Typography fixes with 8 QA gates

7. adapt-skill — Responsive Layout

Breakpoints, touch targets (≥44px), viewport handling, mobile overflow.

Output: Responsive fixes with 9 QA gates

8. optimize-skill — Performance

Bundle size, lazy loading, image optimization, animation compositing, layout thrashing.

Output: Performance fixes targeting Lighthouse ≥90

9. delight-skill — Micro-interactions

Hover/tap feedback, state transitions, loading animation, success/error feedback.

Output: Interaction fixes with 9 QA gates

Typical Flow

  1. critique highlights design gaps
  2. audit finds technical issues
  3. clarify/hard/polish/typeset/adapt/optimize fix them
  4. delight adds the polish

All skills are cross-platform and stack-agnostic.
