### Headline

*Rules that depend on memory fail. Rules that depend on visible blocks succeed.*

That's the insight I learned the hard way building with AI coding agents.

### Body

I spent months watching AI agents produce impressive demos that couldn't survive production. The root cause wasn't capability. It was process.

AI agents are brilliant at generating code. But they have zero built-in discipline:

→ No tests before commit
→ No review before push
→ No memory of rules they agreed to follow 5 minutes ago

I built **Another Agent Skills** to bridge that gap.

→ juandelossantos.github.io/another-agent-skills

An open-source framework that adds a complete development lifecycle to any AI coding agent:

**Define → Plan → Build → Verify → Review → Ship**

What makes it different:

**Harness, not prompts.** 57 skills loaded on demand + 12 pre-commit gates that actually block broken commits. Time-window approval verified by git hooks — no tokens to bypass.

Compatible with Claude Code, Cursor, OpenCode, and any git-based agent. MIT license. Free forever.

If you're building with AI agents in production: what patterns have you found for keeping them disciplined?
